# 论文目标模型结构说明

## 1. 目标

本文件用于明确毕业设计最终模型应当呈现出的结构形态，避免后续论文写作时将“当前实验原型”误写成“最终目标架构”。

核心要求来自 [README.md](/home/easyai/桌面/mamba-yolo3/README.md) 中这句原话：

> 创新性地将擅长局部感知的CNN与擅长长序列建模的Mamba有机融合，构建“空间提取-时序推理”双支路协同的混合骨干网络，同时提升单帧识别精度与帧间关联一致性。

因此，最终论文中的模型不能只是“检测头前加一个时序模块”，而应当是职责清晰的 **双支路时空混合骨干**。

---

## 2. 最终模型应当长什么样

### 2.1 总体结构

最终模型应由以下四部分组成：

1. 空间提取主支路 `Spatial CNN Branch`
2. 时序推理副支路 `Temporal Mamba Branch`
3. 时空融合模块 `Spatio-Temporal Fusion Block`
4. 多尺度检测头 `Detection Head`

对应的数据流应当是：

```text
输入视频片段 {I_(t-k), ..., I_t, ..., I_(t+k)}
        │
        ├── 当前帧 I_t → 空间提取主支路（CNN）
        │                   │
        │                   └── 得到多尺度空间特征 {C3_t, C4_t, C5_t}
        │
        └── 连续帧 clip → 时序推理副支路（Mamba）
                            │
                            └── 得到多尺度时序状态 {T3_t, T4_t, T5_t}

{C3_t, C4_t, C5_t} 与 {T3_t, T4_t, T5_t}
        │
        └── 时空融合模块
                │
                └── 得到融合特征 {F3_t, F4_t, F5_t}
                        │
                        └── 检测头输出当前帧检测结果
```

关键点：

- 检测任务仍然是“当前帧检测”。
- 时序支路服务于当前帧判别，不是输出多帧结果。
- 论文中应强调“检测对象是当前帧，时序信息用于增强当前帧判别”。

---

## 3. 双支路的职责划分

### 3.1 空间提取主支路

职责：

- 负责当前帧局部纹理、边缘、形状、细粒度目标轮廓建模
- 保证单帧检测精度
- 作为主干特征来源，提供稳定的空间表征

建议写法：

- 以 YOLO/Mamba-YOLO 中现有 CNN 风格层级作为主支路
- 在 `C3/C4/C5` 或 `P3/P4/P5` 层输出多尺度空间特征

论文表述建议：

- “空间支路聚焦于局部区域建模，强化小目标边缘响应和局部结构表达能力。”

### 3.2 时序推理副支路

职责：

- 负责连续帧之间的长程依赖建模
- 学习目标在时间维上的运动连续性、显著性变化和上下文稳定性
- 输出与当前帧对齐的时序状态特征

建议写法：

- 输入为相邻帧或短视频片段的特征序列，而不是只输入原图
- 用 Mamba/SSM 单元做状态传递和选择性扫描
- 时序支路输出与空间支路同尺度的状态特征

论文表述建议：

- “时序支路依托 Mamba 的长序列建模优势，显式刻画目标跨帧演化规律与上下文一致性。”

## 4. 融合模块应当如何设计

### 4.1 融合不是简单拼接

论文中的“协同骨干”意味着：

- 不能只在检测头前做一次简单 concat
- 不能只把相邻帧特征平均后加到当前帧
- 需要体现“主支路 + 副支路 + 门控/交互融合”的结构设计

### 4.2 更合理的融合形式

建议采用以下结构之一：

1. 门控融合

```text
G_t = sigmoid(Conv([C_t, T_t]))
F_t = C_t + G_t ⊙ T_t
```

含义：

- 由空间和时序特征共同生成门控权重
- 决定哪些区域更依赖时序补充

2. 残差注入融合

```text
T'_t = Conv(T_t)
F_t = C_t + alpha * T'_t
```

含义：

- 时序支路作为当前帧空间特征的残差补充
- 更稳定，也更容易训练

3. 跨尺度双向融合

```text
F_l = Fuse(C_l, T_l, Up(T_{l+1}), Down(T_{l-1}))
```

含义：

- 不同尺度间的时序状态继续传递
- 更贴合“多尺度状态传递网络”的题目要求

---

## 5. 多尺度状态传递网络应如何落地

README 中要求：

> 在特征金字塔（FPN）中嵌入轻量化 Mamba 单元，增强不同尺度特征间的信息融合，专门优化极小目标的特征表示。

因此最终论文中应体现：

1. 时序状态不是只在单尺度内传播
2. 时序状态可以在 `P3/P4/P5` 之间传递
3. 小目标尺度 `P3` 要得到特别强调

建议结构：

- 在 `P5 → P4 → P3` 的上采样路径中传递时序状态
- 在 `P3 → P4 → P5` 的下采样路径中补充局部稳定性
- 每层配轻量化 Mamba block 或时序门控单元

建议论文描述：

- “通过在 FPN 中引入轻量级状态传递单元，将跨帧动态信息沿多尺度路径传播，增强极小目标的时空表征能力。”

---

## 6. 自适应扫描机制在论文里应如何定义

README 中要求：

> 针对无人机俯拍视角中目标分布稀疏的特点，让 Mamba 动态调整扫描路径，聚焦潜在目标区域。

因此，自适应扫描不能写成泛泛的“用了 Mamba，所以会扫描”。

论文里更合理的定义应当是：

- 利用帧差、显著性响应或历史状态生成引导图 `Guide`
- 用 `Guide` 调节状态更新强度或扫描优先级
- 使模型在运动区域和潜在目标区域投入更多时序建模能力

建议论文公式表达方向：

```text
Guide_t = f(diff_t, saliency_t, prior_t)
State_t = Mamba(X_t, guide=Guide_t)
```

其中：

- `diff_t`：帧差或运动响应
- `saliency_t`：空间显著性
- `prior_t`：可选的历史检测先验或历史状态

## 7. 当前代码与“最终目标架构”的关系

### 7.1 当前代码已经具备的部分

当前项目代码已经具备以下原型基础：

- 三帧输入与时序数据链路
- 多尺度时序融合
- 时序一致性损失
- 自适应稀疏引导原型
- VisDrone-VID / UAVDT 的训练与评测链路

### 7.2 当前代码尚未完全达到的部分

当前代码更接近：

- “在检测头前做时序增强”

尚未完全达到：

- “空间提取-时序推理双支路协同的混合骨干网络”

缺口主要在：

1. 空间主支路与时序副支路的结构边界还不够明确
2. 时序建模还没有深嵌到 backbone/FPN 主路径中
3. 多尺度状态传递的结构表达还不够系统
4. 自适应扫描机制还需要更明确的数学定义与模块封装

因此论文中应严格区分：

- “当前实验原型”
- “毕业设计最终目标架构”

不要把当前实验版本直接表述为最终成熟结构。

---

## 8. 后续代码演化方向

如果后续继续改模型，优先顺序应当是：

1. 明确双支路边界
   - 当前帧 CNN 主支路
   - 连续帧 Mamba 副支路

2. 在 FPN 中加入轻量化时序状态传递单元
   - 强化多尺度状态传播

3. 将融合模块标准化
   - 统一采用门控或残差融合

4. 将自适应扫描机制公式化、模块化
   - 方便论文写作和消融实验

5. 再做速度优化和部署适配

## 9. 论文写作建议表述

如果后续要写论文模型章节，推荐将模型名称和模块名称固定为：

- `Spatial CNN Branch`
- `Temporal Mamba Branch`
- `Adaptive Selective Scan Module`
- `Multi-Scale State Propagation Module`
- `Spatio-Temporal Fusion Block`

模型整体可描述为：

> 本文在 Mamba-YOLO 基础上构建了一个面向无人机视频目标检测的双支路时空混合骨干网络。空间支路利用 CNN 风格层级结构提取当前帧的局部细粒度表征，时序支路利用 Mamba 单元建模连续帧间的长程依赖与动态上下文关系。随后通过门控式时空融合模块在多尺度特征层上进行交互融合，并结合自适应扫描与多尺度状态传递机制，提升小目标检测精度与帧间判别一致性。

---

## 10. 最终一句话总结

毕业设计要求中的这句话，本质上要求的不是“给 Mamba-YOLO 加一个时序模块”，而是：

**把 CNN 和 Mamba 做成职责清晰、能够交互融合的双支路时空混合骨干网络，并在多尺度路径上体现状态传递与自适应扫描。**
